# Telegram → Discord Webhook Forwarder

Автономний скрипт (на базі Telethon/MTProto), який пасивно читає нові пости у вказаних Telegram-каналах та миттєво пересилає їх у ваш Discord сервер за допомогою Webhook. 

Скрипт працює як звичайний акаунт (Userbot), що дозволяє читати **будь-які публічні канали**, навіть якщо у вас немає в них прав адміністратора (на відміну від звичайних ботів через BotFather).

## ⚡ Можливості

- **Пересилка всіх медіа**: Текст, фотографії, відео, документи, голосові та кружечки, стікери — все завантажується і відправляється коректно.
- **Підтримка великих файлів**: Discord webhook дозволяє завантажувати файли до 100 МБ (на бущених серверах). Якщо файл більший (наприклад, 1+ ГБ), бот не впаде, а попередить про це окремим текстом.
- **Динамічний профіль Webhook**: Бот автоматично зчитує назву Telegram-каналу, звідки прийшов пост, і відправляє повідомлення в Discord від його імені (з відповідною аватаркою).
- **Анти-Дублікати (LRU Cache)**: Якщо два канали з вашого списку запостили один і той самий пост (репостнули або виклали ідентичний файл), система розпізнає це і *не створить* подвійного спаму у Discord.

## 🛠 Налаштування (Deploy)

Бот ідеально підходить для хмарного деплою (наприклад, Railway.app, Heroku, Render) завдяки використанню `StringSession`.

### 1. Отримання ключів Telegram API
1. Авторизуйтесь на офіційному порталі [my.telegram.org/apps](https://my.telegram.org/apps).
2. Створіть новий застосунок (дані не мають значення).
3. Скопіюйте ваші **API_ID** та **API_HASH**.

### 2. Отримання токена сесії (Скрипт `login.py`)
Оскільки хмарні хостинги очищують файлову систему між перезапусками, авторизація через файл не підходить. Треба згенерувати текстову сесію у себе на комп'ютері:

1. Встановіть Python та залежності: 
   ```bash
   pip install -r requirements.txt
   ```
2. Створіть файл `.env` та впишіть:
   ```env
   TELEGRAM_API_ID=ваш_id
   TELEGRAM_API_HASH=ваш_hash
   ```
3. Запустіть скрипт: 
   ```bash
   python login.py
   ```
4. Введіть номер телефону і код з Telegram. Скрипт видасть в консоль довгий рядок — це ваш **TELEGRAM_SESSION_STRING**.

### 3. Деплой на Railway (або інший хостинг)
1. Завантажте цей репозиторій на GitHub.
2. Створіть проєкт на Railway та підключіть репозиторій.
3. В налаштуваннях `Variables` додайте ключі:

| Змінна | Опис |
|---|---|
| `TELEGRAM_API_ID` | Ваш API ID |
| `TELEGRAM_API_HASH` | Ваш API Hash |
| `TELEGRAM_SESSION_STRING` | Згенерований рядок авторизації з кроку 2 |
| `DISCORD_WEBHOOK_URL` | URL вашого вебхуку з налаштувань Discord |

Скрипт автоматично встановить залежності та почне слухати канали!

## 📂 Структура проєкту

- `bot.py` — Головний скрипт, обробник подій Telegram та POST-відправник.
- `login.py` — Утиліта для генерації `TELEGRAM_SESSION_STRING`.
- `requirements.txt` — Залежності: Telethon (MTProto), httpx (асинхронні запити), python-dotenv (робота з .env).
- `railway.toml` / `Procfile` — Інструкції старту для хмарних платформ.
- `.gitignore` — Захист від випадкового зливу ключів на GitHub.
